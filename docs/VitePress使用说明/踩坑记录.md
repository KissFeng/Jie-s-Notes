# 踩坑记录

搭建 VitePress + GitHub Actions 自动部署到云服务器的完整过程中，遇到的所有坑。

---

## 1. VitePress 初始化向导无法自动化

**问题**：`npx vitepress init` 是交互式向导，通过管道传参会乱码，无法用脚本自动化。

**解决**：跳过向导，手动创建以下文件：

```
.vitepress/config.mts    # 全局配置
index.md                  # 首页
markdown-examples.md      # 示例页
api-examples.md           # 示例页
```

同时在 `package.json` 中手动添加脚本：

```json
{
  "scripts": {
    "dev": "vitepress dev",
    "build": "vitepress build",
    "preview": "vitepress preview"
  }
}
```

---

## 2. 侧边栏默认展开问题

**问题**：VitePress 侧边栏分组默认是展开的，分类多了页面很乱。

**解决**：在 `.vitepress/config.mts` 中设置 `collapsed: true`：

```ts
sidebar: [
  {
    text: '分类名称',
    collapsed: true,  // 默认收起
    items: [...]
  }
]
```

---

## 3. ssh-deploy 不支持密码认证

**问题**：GitHub Actions 使用 `easingthemes/ssh-deploy` 时，配置了 `PASSWORD` 参数报错：

```
❌ [INPUTS] sshPrivateKey is mandatory
Warning: Unexpected input(s) 'PASSWORD', valid inputs are [...]
```

**原因**：该 action 已移除密码认证，只支持 SSH 私钥。

**解决**：

1. 本地生成密钥对：
   ```bash
   ssh-keygen -t ed25519 -C "github-actions" -f ~/.ssh/github_actions -N ""
   ```

2. 把公钥传到服务器：
   ```bash
   cat ~/.ssh/github_actions.pub | ssh -p 26463 root@服务器IP "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
   ```

3. 在 GitHub 仓库 **Settings → Secrets** 中添加 `SSH_PRIVATE_KEY`，值为私钥全部内容。

---

## 4. 自定义 SSH 端口配置

**问题**：云服务器 SSH 端口不是默认的 22，而是 26463，GitHub Actions 连接不上。

**解决**：通过 GitHub Secret 配置端口，不要硬编码在 workflow 中：

- 新增 Secret：`SERVER_PORT` = `26463`
- workflow 中通过 `${{ secrets.SERVER_PORT }}` 引用

---

## 5. rsync 部署成功但服务器目录为空（最大坑）

**问题**：GitHub Actions 日志显示绿色 ✅，rsync 输出 `[DONE]`，但服务器目标目录是空的，nginx 返回 403 Forbidden。

**排查过程**：
- 本地 `npm run build` 构建产物正常
- SSH 连接服务器正常
- 目标目录 `/www/wwwroot/jie-docs` 存在且有写入权限
- 手动 rsync 也失败，文件就是传不过去

**原因**：GitHub Actions Ubuntu 环境中的 rsync 版本与服务器 rsync 存在协议兼容问题，传输静默失败。

**解决**：放弃 `ssh-deploy`（rsync），改用 `scp` 直接传输：

```yaml
- name: Deploy to Server
  env:
    SSH_PRIVATE_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
    REMOTE_HOST: ${{ secrets.SERVER_HOST }}
    REMOTE_USER: ${{ secrets.SERVER_USER }}
    REMOTE_PORT: ${{ secrets.SERVER_PORT }}
  run: |
    mkdir -p ~/.ssh
    echo "$SSH_PRIVATE_KEY" > ~/.ssh/deploy_key
    chmod 600 ~/.ssh/deploy_key
    ssh -p $REMOTE_PORT -i ~/.ssh/deploy_key -o StrictHostKeyChecking=no $REMOTE_USER@$REMOTE_HOST "mkdir -p /www/wwwroot/jie-docs"
    scp -P $REMOTE_PORT -i ~/.ssh/deploy_key -o StrictHostKeyChecking=no -r .vitepress/dist/* $REMOTE_USER@$REMOTE_HOST:/www/wwwroot/jie-docs/
```

## 6. nginx 403 Forbidden

**问题**：访问网站返回 403。

**原因**：服务器目标目录为空，nginx 没有 `index.html` 可以返回。

**解决**：确保文件正确传输后，nginx 配置如下即可：

```nginx
server {
    listen 9999;
    server_name _;

    location / {
        root /www/wwwroot/jie-docs;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
```

> ⚠️ 如果自定义了端口（如 9999），访问时必须带端口号：`http://服务器IP:9999`

---

## 7. GitHub Actions Node.js 版本警告

**问题**：日志中出现警告：

```
Warning: Node.js 20 actions are deprecated.
```

**原因**：`actions/checkout@v4` 和 `actions/setup-node@v4` 默认使用 Node.js 20，GitHub 将在 2026 年 9 月移除。

**解决**：目前不影响使用，等官方发布 v5 版本后升级即可。也可以在 workflow 中添加环境变量提前适配：

```yaml
env:
  FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: "true"
```

## 总结

| 坑 | 耗时程度 | 核心解法 |
|----|---------|---------|
| 初始化向导不支持自动化 | ⭐ | 手动创建文件 |
| ssh-deploy 不支持密码 | ⭐⭐ | 改用 SSH 私钥 |
| 自定义 SSH 端口 | ⭐ | 配置 Secret |
| rsync 静默失败 | ⭐⭐⭐⭐⭐ | 改用 scp |
| nginx 403 | ⭐⭐ | 确认文件已传输 |
