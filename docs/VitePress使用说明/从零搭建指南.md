# 从零搭建 VitePress 文档站并部署到云服务器

完整记录从本地初始化到 GitHub Actions 自动部署的每一步。

---

## 一、环境准备

| 工具 | 最低版本 | 说明 |
|------|---------|------|
| Node.js | 18+ | 推荐 22 LTS |
| Git | 2.x | 版本控制 |
| 云服务器 | 任意 Linux | 需有 SSH 访问权限 |
| GitHub 账号 | - | 用于托管代码和 CI/CD |

---

## 二、本地初始化 VitePress

### 1. 创建项目目录

```bash
mkdir MyVitePress && cd MyVitePress
npm init -y
```

### 2. 安装 VitePress

```bash
npm install -D vitepress
```

### 3. 创建基础文件结构

VitePress 的交互式向导（`npx vitepress init`）无法脚本化，直接手动创建：

```
MyVitePress/
├── .vitepress/
│   └── config.mts        # 全局配置
├── docs/                  # 文档目录（可选，也可放根目录）
│   └── ...
├── index.md               # 首页
├── markdown-examples.md   # Markdown 示例
└── package.json
```

### 4. 配置 package.json 脚本

```json
{
  "scripts": {
    "dev": "vitepress dev",
    "build": "vitepress build",
    "preview": "vitepress preview"
  }
}
```

### 5. 编写首页 index.md

```md
---
layout: home
hero:
  name: "Jie's Notes"
  text: "我的个人公开知识库"
  tagline: 记录学习、分享技术
  actions:
    - theme: brand
      text: 开始阅读
      link: /docs/
---
```

### 6. 编写全局配置 .vitepress/config.mts

```ts
import { defineConfig } from 'vitepress'

export default defineConfig({
  title: "Jie's Notes",
  description: '我的个人公开知识库',
  themeConfig: {
    // 顶部导航
    nav: [
      { text: '首页', link: '/' },
      { text: '示例', link: '/markdown-examples' }
    ],

    // 侧边栏
    sidebar: [
      {
        text: '分类名称',
        collapsed: true,   // 默认收起
        items: [
          { text: '总览', link: '/docs/分类/' }
        ]
      }
    ],

    // 社交链接
    socialLinks: [
      { icon: 'github', link: 'https://github.com/你的用户名' }
    ]
  }
})
```

### 7. 本地预览

```bash
npm run dev
```

浏览器打开 `http://localhost:5173` 查看效果。

## 三、目录组织规范

```
docs/
├── 分类A/
│   ├── index.md          # 分类总览页
│   └── 文章1.md
├── 分类B/
│   ├── index.md
│   └── 文章2.md
└── ...
```

每个分类文件夹对应侧边栏的一个分组，`index.md` 作为总览页。在 `config.mts` 的 `sidebar` 中按相同结构配置即可。

## 四、GitHub 仓库与推送

### 1. 初始化 Git

```bash
git init
echo "node_modules/\n.vitepress/dist/\n.vitepress/cache/" > .gitignore
git add .
git commit -m "feat: 初始化 VitePress 项目"
```

### 2. 创建远程仓库并推送

在 GitHub 上创建仓库（如 `Jie-s-Notes`），然后：

```bash
git remote add origin git@github.com:你的用户名/你的仓库名.git
git branch -M main
git push -u origin main
```

## 五、云服务器配置（nginx）

SSH 登录服务器，配置 nginx 反向代理：

```bash
# 创建部署目录
mkdir -p /www/wwwroot/jie-docs

# 编辑 nginx 配置
vi /etc/nginx/conf.d/jie-docs.conf
```

写入以下内容：

```nginx
server {
    listen 9999;
    server_name _;

    location / {
        root /www/wwwroot/jie-docs;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
```

重载 nginx：

```bash
nginx -t && nginx -s reload
```

> 如果使用宝塔面板，直接在网站管理中创建站点，指定目录和端口即可。

## 六、配置 GitHub Actions 自动部署

### 1. 生成 SSH 密钥对

本地终端执行：

```bash
ssh-keygen -t ed25519 -C "github-actions" -f ~/.ssh/github_actions -N ""
```

### 2. 上传公钥到服务器

```bash
cat ~/.ssh/github_actions.pub | ssh -p 你的SSH端口 root@服务器IP "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

### 3. 配置 GitHub Secrets

进入仓库 **Settings → Secrets and variables → Actions**，添加：

| Secret 名称 | 值 |
|-------------|---|
| `SSH_PRIVATE_KEY` | `~/.ssh/github_actions` 私钥的**全部内容** |
| `SERVER_HOST` | 服务器 IP |
| `SERVER_USER` | SSH 用户名（如 `root`） |
| `SERVER_PORT` | SSH 端口（如 `26463`） |

### 4. 编写 workflow 文件

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy VitePress to Cloud Server

on:
  push:
    branches:
      - main

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'

      - name: Install Dependencies and Build
        run: |
          npm install
          npm run build

      - name: Deploy to Server
        env:
          SSH_PRIVATE_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
          REMOTE_HOST: ${{ secrets.SERVER_HOST }}
          REMOTE_USER: ${{ secrets.SERVER_USER }}
          REMOTE_PORT: ${{ secrets.SERVER_PORT }}
        run: |
          mkdir -p ~/.ssh
          echo "$SSH_PRIVATE_KEY" > ~/.ssh/deploy_key
          chmod 600 ~/.ssh/deploy_key
          tar czf - -C .vitepress/dist . | ssh -p $REMOTE_PORT -i ~/.ssh/deploy_key -o StrictHostKeyChecking=no $REMOTE_USER@$REMOTE_HOST "mkdir -p /www/wwwroot/jie-docs && tar xzf - -C /www/wwwroot/jie-docs"
```

### 5. 提交触发部署

```bash
git add .
git commit -m "ci: 添加 GitHub Actions 自动部署"
git push origin main
```

推送后 GitHub Actions 自动触发：安装依赖 → 构建 → 打包上传到服务器。

## 七、验证

1. 打开 GitHub 仓库的 **Actions** 页面，确认 workflow 运行成功（绿色 ✅）
2. 浏览器访问 `http://服务器IP:端口`，看到首页即部署成功

## 八、日常使用流程

写文档 → 本地预览 → 提交推送 → 自动部署，整个流程：

```bash
# 1. 本地开发预览
npm run dev

# 2. 新增或编辑 docs/ 下的 Markdown 文件

# 3. 提交并推送，自动部署
git add .
git commit -m "content: 新增文章标题"
git push origin main
```

大约 1~2 分钟后刷新线上页面即可看到更新。
