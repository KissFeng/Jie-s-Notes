# Claude 封号这波操作，直接把我三观干碎了

说真的，这两天群里全在哀嚎，AI 圈简直像是被炸过一样。大家都在吐槽自己的 Claude 账号莫名其妙被封了。很多老哥平时稳得一批，用了一年多的号，这次竟然也全翻车了，有些搞初创的公司甚至连整个账号池都被连锅端。

老玩家都知道，Anthropic（也就是大家常喷的某A厂）对国内访问卡得一向很严。但之前大家都挺纳闷的：我明明挂了海外的梯子，IP 都飞到洛杉矶了，它到底是长了什么天眼，能隔着网线精准定位到我是国内用户的？

直到昨天，国外有个搞逆向工程的大哥闲着没事，把官方出的终端编程工具 Claude Code 直接给反编译了，底裤扒得一干二净。看完大佬的分析我整个人都不好了——这哪是简单的封锁啊，根本就是一场处心积虑的“无间道”，他们在客户端里偷偷埋了个超级恶心的追踪系统。

## 专坑代理用户的“钓鱼”

最恶心的是，这套机制还有个专门的触发条件。只要你在电脑里配了 `ANTHROPIC_BASE_URL` 这个环境变量，也就是用了中转 API，这套追踪系统就会立刻激活。

你想啊，咱们国内开发者哪有几个是直连官方 API 的？几乎所有人都要走代理和中转。所以这东西说白了，就是专门为咱们国内用户“量身定制”的陷阱。

它具体是怎么查水表的呢？主要有两招，特别鸡贼。

第一招就是看你电脑的时区。只要系统时区是 `Asia/Shanghai` 或者乌鲁木齐时区，它就直接给你打个嫌疑人标签。咱们虽然天天挂代理，但谁没事干去改系统时间啊？这招一出，几乎99%的国内用户全中招了。

第二招更离谱，比对你的中转域名。它内置了一份加密的黑名单，里面有140多个域名。逆向大佬解开一看，好家伙，.cn 结尾的域名全部腰斩，百度、阿里、字节这些国内大厂一个没落，甚至连人家公司内部测试的域名都在上面！只要你的代理地址踩中这些关键词，恭喜你，直接被死死盯上。

## 连黑客看了都直呼内行的“隐写术”

如果说前面这些还只是常规手段，那接下来它上传数据的方式，简直让我大开眼界，甚至有点背脊发凉。

它查出你身份之后，是怎么把消息偷偷传回总部的？

你在终端里每次敲下回车向 Claude 提问时，它都会在最前面偷偷加一句系统提示词。里面有一行特别正常的日期信息，类似 `Today's date is 2026-06-30.`

重点来了！在发出去的最后一秒，它会对这行字做手脚。
如果它发现你是中国时区，就会把日期里的横杠 `-` 变成斜杠 `/`。
而且，根据你踩中黑名单的程度，它还会偷偷把你提示词里的单引号 `'` 替换成另外三种 Unicode 字符（比如修饰字母撇号）。

这四个不同的单引号，在咱们的屏幕上长得一模一样！肉眼根本看不出任何区别！但在机器端，它的编码是完全不同的。A厂的服务器收到请求后，只要扫一眼你是横杠还是斜杠、单引号是哪个字符，瞬间就能知道“哦，这是个中国用户，而且用了某某中转站”。

这种玩法在安全圈叫“隐写术”。最可怕的是，这整个过程根本不需要偷偷发什么奇怪的网络请求，也没有任何可疑流量，连抓包软件都查不出猫腻，因为它就藏在你正常的提问里！

## 就挺心寒的

那个逆向出这段代码的老外直接在网上开喷了，说这简直就是间谍软件的行为。

有人洗地说是为了防滥用。但问题是，Claude Code 可是个跑在你本地电脑上的终端程序啊！它有权限读你的代码、翻你的文件夹、甚至执行 Shell 命令。你给了它这么高的信任，结果它居然天天在你眼皮子底下玩这种地下特工的把戏，不仅没在更新日志里提过半个字，还把私货藏得这么深。

今天为了封号能收集你的时区，明天谁敢保证它不会把你的私密代码也偷偷传回去？一家天天把“AI 安全”、“透明度”挂在嘴边的公司，背地里却搞这种偷偷摸摸的操作，被骂双标真的活该。

而且最搞笑的是，真正那些搞灰产、大批量倒卖账号的人，改个时区换个域名就能轻松绕过去。最后倒霉的，全是我们这些老老实实写代码的普通开发者。

现在想好好写个代码，不仅要防封号、防限流，还要防这种神出鬼没的隐写术，心真是太累了。
