# 域名邮箱设置指南

如何用你自己的域名（如 `kissfeng@power8.top`）收发邮件，打造专属的个人品牌邮箱。

## 为什么需要域名邮箱？

- **专业感**：`kissfeng@yourdomain.com` 比 `xxx@gmail.com` 看起来更正式
- **隐私隔离**：不同服务注册不同前缀（`github@`、`steam@`），垃圾邮件一眼可辨来源
- **品牌统一**：个人网站、简历、名片用同一个域名，辨识度高

## 核心方案对比

| 方案 | 成本 | 发信 | 收信 | 适用场景 |
|------|------|------|------|----------|
| **Cloudflare Email Routing** | 完全免费 | 需借助 Gmail 代发 | ✅ 无限前缀 | 个人日常、技术爱好者 |
| **飞书 / 钉钉企业邮箱** | 免费 | ✅ 独立发信 | ✅ 独立收信 | 国内为主、需要独立客户端 |
| **Zoho Mail** | 免费（5 用户） | ✅ 独立发信 | ✅ 仅网页端 | 海外为主 |
| **iCloud+ 自定义域名** | ¥6/月起 | ✅ 原生体验 | ✅ 系统邮件 App | 苹果生态用户 |
| **Google Workspace** | $6/月起 | ✅ 专业级 | ✅ 完整体验 | 企业/商业用途 |

## 方案一：Cloudflare Email Routing（推荐）

> 完全免费、无限前缀、Catch-all 兜底，是个人用户的性价比天花板。

### 前提条件

- 一个你拥有的域名（如在阿里云、腾讯云、GoDaddy 购买的）
- 域名需要托管到 Cloudflare（即 DNS 解析交给 Cloudflare 管理）

### 第一步：将域名托管到 Cloudflare

如果你的域名不在 Cloudflare，需要先迁移 DNS 托管：

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com)，点击 **Add a site**
2. 输入你的域名，Cloudflare 会自动扫描现有 DNS 记录
3. Cloudflare 会分配给你两个 Nameserver 地址（如 `xxx.ns.cloudflare.com`）
4. 去你的域名注册商后台（如腾讯云、阿里云），找到 **DNS 服务器** 设置
   - ⚠️ 注意：不是"解析记录"页面，而是"域名管理"页面里的"修改 DNS 服务器"
   - 腾讯云：控制台 → 我的域名 → 管理 → 修改 DNS 服务器
   - 阿里云：域名控制台 → 管理 → DNS 修改
5. 将原来的 DNS 服务器地址替换为 Cloudflare 提供的两个地址
6. 回到 Cloudflare 点击 **Check nameservers**，等待生效（通常 5~15 分钟）

> 💡 生效后域名主页会显示绿色勾，提示托管成功。

### 第二步：配置 Email Routing 收信

域名托管成功后，在 Cloudflare 左侧菜单找到 **Email → Email Routing**。

#### 2.1 添加目标收件邮箱

1. 切换到 **Destination addresses** 标签页
2. 点击 **Add destination address**
3. 输入你常用的私人邮箱（如 `xxx@gmail.com`）
4. 去该邮箱查收 Cloudflare 的验证邮件，点击激活链接

#### 2.2 创建邮箱前缀

1. 切换到 **Routing rules** 标签页
2. 点击 **Create routing rule**
3. 填写：
   - **Email pattern** 左侧：填入你想要的前缀（如 `kissfeng`）
   - **Email pattern** 右侧：选择你的域名（如 `power8.top`）
   - **Action**：选择 **Send to an email**
   - **Destination address**：选择你验证过的私人邮箱
4. 点击 **Save**

> 这样你就拥有了 `kissfeng@power8.top` 这个专属邮箱！

#### 2.3 开启 Catch-all（强烈推荐）

Catch-all 是 Cloudflare Email Routing 的杀手级功能：**任何前缀都会被自动转发**，你不需要提前创建规则。

1. 在 **Routing rules** 页面找到底部的 **Catch-all** 规则
2. 点击右侧 `...` → **Edit**
3. 将 Action 改为 **Send to an email**，目标地址选你的私人邮箱
4. 开启开关变为 **Enabled**

开启后，随便什么前缀（`github@power8.top`、`steam@power8.top`、`anything@power8.top`）都能收到！

#### 2.4 自动配置 DNS 记录

配置完成后，Cloudflare 会提示需要添加 MX 和 TXT 记录。

直接点击 **Add records automatically**，Cloudflare 会一键写入所有需要的 DNS 记录：

- **MX 记录**：告诉互联网谁帮你收邮件
- **TXT 记录（SPF）**：防伪标记，证明服务器有权替你的域名发信

看到状态变为 **Routing（正在路由）** 就说明收信能力已开通。

### 第三步：用域名邮箱对外发信（Gmail 代发）

Cloudflare Email Routing 只负责收信。要以 `kissfeng@power8.top` 的身份发信，需要借助 Gmail 的免费 SMTP 服务。

#### 3.1 生成 Google 应用专用密码

1. 登录 [myaccount.google.com](https://myaccount.google.com)
2. 左侧 → **安全性** → 确保已开启 **两步验证**
3. 在两步验证页面底部找到 **应用密码**
4. 创建一个新密码（名称随意，如 `CF-Email`）
5. 复制生成的 16 位密码（只会出现一次，务必保存好）

#### 3.2 在 Gmail 中添加域名邮箱

1. 打开网页版 Gmail → 右上角齿轮 → **查看所有设置**
2. 切换到 **账号和导入** 标签页
3. 找到 **查看其他电子邮箱地址** → 点击 **添加其他电子邮箱地址**
4. 在弹出窗口中填写：
   - **姓名**：你想让别人看到的昵称
   - **电子邮件地址**：你的域名邮箱（如 `kissfeng@power8.top`）
   - ⚠️ **取消勾选** "视作别名"
5. 点击 **下一步**

#### 3.3 配置 SMTP 发信服务器

| 参数 | 值 |
|------|-----|
| SMTP 服务器 | `smtp.gmail.com` |
| 端口 | `587` |
| 用户名 | 你的 Gmail 地址（如 `xxx@gmail.com`） |
| 密码 | 刚才生成的 16 位应用专用密码 |
| 连接类型 | TLS |

填写完成后点击 **添加账号**。

#### 3.4 验证

Gmail 会向 `kissfeng@power8.top` 发一封验证邮件。因为 Cloudflare 转发已生效，这封信会秒级出现在你的 Gmail 收件箱中。打开邮件，输入验证码或点击验证链接即可。

> 🎉 配置完成后，Gmail 写信时的"发件人"会变成下拉菜单，可以自由切换 `xxx@gmail.com` 和 `kissfeng@power8.top`。

### 第四步：修复 SPF 记录（消除代发警告）

使用 Gmail 代发时，QQ 邮箱等服务可能会显示"发件人无法验证"的红色警告。原因是 SPF 记录只信任了 Cloudflare，没有包含 Google。

#### 修改方法

1. 进入 Cloudflare → **DNS → Records**
2. 找到类型为 **TXT**、名称为 **@** 的 SPF 记录
3. 将内容从：

```
v=spf1 include:_spf.mx.cloudflare.net ~all
```

修改为：

```
v=spf1 include:_spf.mx.cloudflare.net include:_spf.google.com ~all
```

4. 保存，等待 5~10 分钟 DNS 生效
5. 再次发送测试邮件，警告消失

> 💡 这段记录的意思是告诉全球邮件服务器：「只有 Cloudflare 和 Google 可以代我发信，其他都是冒牌货。」

## 方案二：飞书企业邮箱（国内推荐）

如果你需要纯净的独立发信（无代发痕迹），且主要和国内联系，飞书企业邮箱是最佳免费方案。

### 优势

- 完全独立，发件人干净，无"代发"字样
- 免费支持约 50 个成员
- 国内网络友好，送达率 100%
- 支持飞书 App 或通过 IMAP/SMTP 挂载到系统邮件客户端

### 配置步骤

1. 注册飞书企业/组织账号（个人创建组织即可，不需要营业执照）
2. 飞书后台 → 企业邮箱 → 输入你的域名
3. 飞书会给你几条 MX 和 TXT 记录
4. 回到 Cloudflare DNS，**删除** 之前 Cloudflare Email Routing 的 MX 记录，替换为飞书的
5. 等待 DNS 生效后即可在飞书中收发邮件

## 方案三：iCloud+ 自定义邮件域名（苹果用户）

如果你是 iPhone / Mac 用户，且已订阅 iCloud+（最低档 ¥6/月）：

1. 进入 iPhone **设置 → Apple ID → iCloud → 自定义电子邮件域名**
2. 添加你的域名并按照指引配置 DNS 记录
3. 配置完成后，系统自带"邮件" App 可以直接用域名邮箱收发
4. 走苹果官方服务器，无代发，干净且原生

## 进阶：通过 API / 开源项目管理邮箱前缀

如果你想通过程序或网页面板来动态创建、删除邮箱前缀，而不是每次都登 Cloudflare 后台手动操作，有以下选择：

### Cloudflare 官方 API

只要你有 API Token，就可以直接调用 Email Routing API：

**获取认证信息：**

- `CLOUDFLARE_API_TOKEN`：Cloudflare 后台 → 个人资料 → API 令牌 → 创建一个有 Email Routing 权限的 Token
- `ZONE_ID`：域名 Overview 主页右下角可以找到

**创建邮箱前缀：**

```bash
curl -X POST "https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/email/routing/rules" \
  -H "Authorization: Bearer {CLOUDFLARE_API_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "actions": [{ "type": "forward", "value": ["your@gmail.com"] }],
    "matchers": [{ "field": "to", "type": "literal", "value": "kissfeng@power8.top" }],
    "name": "My Rule",
    "enabled": true
  }'
```

**列出所有规则：**

```bash
curl "https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/email/routing/rules" \
  -H "Authorization: Bearer {CLOUDFLARE_API_TOKEN}"
```

**删除规则：**

```bash
curl -X DELETE "https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/email/routing/rules/{RULE_ID}" \
  -H "Authorization: Bearer {CLOUDFLARE_API_TOKEN}"
```

### 开源项目推荐

- **[agentic-inbox](https://github.com/cloudflare/agentic-inbox)**：Cloudflare 官方开源，结合 Workers + D1 + Workers AI，支持 AI 智能分类、自动别名管理、邮件摘要，可以部署在 Cloudflare Pages 上零成本运行
- **[vmail](https://github.com/oiov/vmail)**：基于 Cloudflare Workers + D1 的临时邮箱/域名邮箱管理面板，支持网页实时收信、前缀管理，同样零服务器成本

### Cloudflare Workers 进阶玩法

在 Email Routing 中，除了转发到邮箱，还可以选择 **Send to a Worker**。你可以写几行 JavaScript 拦截邮件，解析后通过 Webhook 推送到飞书、钉钉或微信，实现「不开邮箱也能看信」。

## 常见问题

### Cloudflare Email Routing 发信会被认出来吗？

部分严格的服务（如企业 Outlook）查看邮件原文时会看到「由 xxx@gmail.com 代发」。日常使用（个人邮件、注册验证）完全没问题。如果介意，可以改用飞书企业邮箱或 Resend 等专业发信服务。

### Catch-all 有什么安全隐患吗？

Catch-all 意味着任何人都能往你的域名发邮件，包括随机前缀。好处是注册各种服务时可以用任意前缀；坏处是垃圾邮件发送者也可能猜到你的域名在发信。实际使用中，Cloudflare 的反垃圾能力不错，问题不大。

### 如何选择？

| 你的需求 | 推荐方案 |
|---------|---------|
| 想白嫖、够用就行 | Cloudflare Email Routing |
| 不想看到"代发"、主要国内联系 | 飞书企业邮箱 |
| 苹果用户、愿意花 ¥6/月 | iCloud+ 自定义域名 |
| 企业/商业用途、需要日历等功能 | Google Workspace / Microsoft 365 |
